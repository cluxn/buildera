<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser (buildera.co) → Laravel API | CORS enforced via config/cors.php; only FRONTEND_URL origin allowed for api/* |
| Next.js server → POST /api/leads | X-API-Key header required; key validated against APP_API_KEY env var (implemented Phase 2) |
| .env file → filesystem | .env in .gitignore; APP_KEY never committed; .env.example has only placeholders |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | .env / APP_KEY | mitigate | .gitignore must include .env; .env.example APP_KEY value is placeholder text, not a real base64 key |
| T-02-02 | Elevation of Privilege | CORS wildcard | mitigate | allowed_origins reads from FRONTEND_URL env var — never uses '*'; supports_credentials: false prevents cookie-based CSRF from cross-origin |
| T-02-03 | Spoofing | X-API-Key header in CORS | mitigate | X-API-Key listed in allowed_headers so preflight OPTIONS passes; actual key validation added in Phase 2 ApiKeyMiddleware |
| T-02-04 | Information Disclosure | Filament /admin path | accept | /admin requires authentication (login page); no sensitive data exposed to unauthenticated requests |
| T-02-SC | Tampering | composer package installs | mitigate | All packages in Package Legitimacy Audit table in 01-RESEARCH.md marked Approved; no [ASSUMED] or [SUS] packages |
</threat_model>
