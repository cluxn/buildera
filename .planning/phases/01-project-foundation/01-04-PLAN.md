<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Monitoring service → GET /api/health | No auth required — endpoint is public by design; returns only status/db/timestamp — no sensitive data |
| AppServiceProvider → .env | Validates env vars on every boot; RuntimeException prevents the app from serving requests with missing config |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | GET /api/health response | accept | Response reveals only status/db/timestamp — no stack traces, no app version, no path info. DB 'error' string does not leak connection details. |
| T-04-02 | Denial of Service | GET /api/health — no rate limit | accept | Health check is read-only, makes one DB ping, returns ~100 bytes JSON. Throttle added if abuse observed at deploy time (Phase 10). |
| T-04-03 | Tampering | AppServiceProvider env validation | accept | Validation runs on every boot — correct behavior. An attacker cannot exploit this; it only affects legitimate deployments with misconfigured .env. |
| T-04-04 | Information Disclosure | error.tsx shows error.message | mitigate | error.message in production Next.js is sanitized by the framework — React's error boundary receives a generic "minified React error" message in production builds, not the raw internal error. No server internals are exposed. |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new package installs in this plan — only source files created/modified |
</threat_model>
