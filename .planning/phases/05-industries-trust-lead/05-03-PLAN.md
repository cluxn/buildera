<api_contract>
POST /api/leads — already built in Phase 2 (plan 02-03):
- Headers: X-API-Key: {NEXT_PUBLIC_API_KEY}, Content-Type: application/json
- Body: { name, email, phone?, company?, message?, source_form, utm_source?, utm_medium?, utm_campaign? }
- Throttle: 5/hr per IP — show friendly error on 429
- Honeypot: include empty `website` field in body (Laravel ignores non-empty honeypot and silently drops lead)
- 24hr duplicate detection: 422 response — show "We already have your enquiry, we'll be in touch soon!"
- Success: 201 response → redirect to /thank-you

fetchSettings() returns: { calendly_url: string, whatsapp_number: string, ... } (all fields in SETTINGS_FALLBACK)

NEXT_PUBLIC_API_URL — base URL from .env.local
NEXT_PUBLIC_API_KEY — X-API-Key value from .env.local (public, safe to use in browser)
</api_contract>

    - Renders Calendly inline widget via `<Script>` approach OR direct iframe approach.
    - Use the direct iframe approach (simpler, no external script dependency):
      ```tsx
      "use client"
      export function CalendlyEmbed({ calendlyUrl }: { calendlyUrl: string }) {
        if (!calendlyUrl) return (
          <div className="flex items-center justify-center h-64 bg-[var(--brand-surface)] rounded-xl border border-border">
            <p className="text-muted-foreground text-sm">Booking calendar coming soon. Please email info@buildera.co to schedule a call.</p>
          </div>
        )
        const embedUrl = calendlyUrl.includes('?')
          ? `${calendlyUrl}&embed_type=Inline&hide_event_type_details=1`
          : `${calendlyUrl}?embed_type=Inline&hide_event_type_details=1`
        return (
          <div className="rounded-xl overflow-hidden border border-border" style={{ minHeight: 700 }}>
            <iframe
              src={embedUrl}
              width="100%"
              height="700"
              frameBorder="0"
              title="Schedule a call with Buildera"
              loading="lazy"
            />
          </div>
        )
      }
      ```
    - If calendlyUrl is empty string, show fallback message (email link).
