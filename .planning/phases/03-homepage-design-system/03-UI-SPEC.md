## Spacing Scale

All values are multiples of 4. Used as Tailwind utility values (e.g., `p-4` = 16px in Tailwind 4).

| Token | px Value | Tailwind Class | Usage |
|-------|----------|----------------|-------|
| xs | 4px | `gap-1`, `p-1` | Icon internal gaps, badge padding |
| sm | 8px | `gap-2`, `p-2` | Compact inline padding, icon-to-text gaps |
| md | 16px | `gap-4`, `p-4` | Default card padding, form field gaps |
| lg | 24px | `gap-6`, `p-6` | Card padding (desktop), nav item spacing |
| xl | 32px | `gap-8`, `p-8` | Section internal padding (mobile) |
| 2xl | 48px | `gap-12`, `py-12` | Section padding (desktop top/bottom) |
| 3xl | 64px | `gap-16`, `py-16` | Major section breaks (hero, CTA) |
| 4xl | 96px | `py-24` | Hero section vertical padding |

**Exceptions:**
- Touch targets (mobile): minimum 48px height/width on all interactive elements (A11Y-02). Use `min-h-[48px]` on buttons and nav items.
- Mega dropdown panel: full-width, no horizontal padding constraint — constrained by inner `container` class only.
- Stat card numbers: 64px font size (display value), not spacing.
- Hero section: `min-h-screen` (100dvh), not a spacing token.

**Section horizontal padding:** `px-4` (mobile) → `px-6` (sm) → `container mx-auto px-8` (lg+). Max content width: `max-w-7xl` (1280px).

## Animation Contract

### Global Motion Defaults

Motion library: `import { motion } from 'motion/react'`
Scroll trigger: `whileInView` + `viewport={{ once: true, margin: "-100px" }}`
Performance: `will-change: transform, opacity` on animated elements. Remove after animation completes via `onAnimationComplete`.
Reduced motion: All animations must respect `prefers-reduced-motion`. Use `useReducedMotion()` from motion/react to gate animations.

**Base scroll reveal (used by default on all section blocks):**
```tsx
initial={{ opacity: 0, y: 24 }}
whileInView={{ opacity: 1, y: 0 }}
viewport={{ once: true, margin: "-100px" }}
transition={{ duration: 0.5, ease: "easeOut" }}
```

**Grid stagger (applied to card grids):**
Each child card gets `transition={{ delay: index * 0.08 }}` added to the base reveal transition. 80ms per item.

**Hover card (universal, all cards):**
```tsx
whileHover={{ y: -4 }}
transition={{ duration: 0.2, ease: "easeOut" }}
/* Plus CSS: box-shadow: 0 0 0 2px var(--brand-primary) on hover */
```

**Spring easing (for counter fills and ring draws):**
```tsx
transition={{ type: "spring", stiffness: 100, damping: 20 }}
```

#### Section 2: Stats Bar

**Narrative purpose:** "Prove capability with numbers before the visitor reads a single word of copy." Numbers count up like a slot machine resolving on truth.

**Counter animation (each of 4 stats):**
```tsx
// Use motion useMotionValue + useTransform + useSpring
// Trigger: whileInView + viewport={{ once: true }}
// Implementation:
const count = useMotionValue(0)
const spring = useSpring(count, { stiffness: 60, damping: 20 })
const rounded = useTransform(spring, (v) => Math.round(v))

useEffect(() => {
  if (inView) count.set(targetValue) // e.g., 150 for "150+"
}, [inView])
```
- Duration equivalent: ~1.5s spring settle
- Stagger: `delay: index * 0.2` (200ms between each of the 4 stats)
- Easing: spring (stiffness: 60, damping: 20) — decelerates naturally at target

**Container entrance:** Stats bar `div` slides up with `initial={{ opacity: 0, y: 16 }}` on scroll-into-view before counters start.

#### Section 4: Solutions Grid

**Narrative purpose:** "Show the specific problems Buildera solves." Animation visualizes problems being illuminated one by one — a grid lighting up from bottom-left to top-right, like a board being solved.

**Diagonal cascade (bottom-left to top-right):**

## `"use client"` Boundary Map

Strict rule from CLAUDE.md: `"use client"` ONLY on components that require browser APIs or React state/effects.

| Component | Needs `"use client"` | Reason |
|-----------|---------------------|--------|
| `SiteNavClient` | YES | scroll event listener for glass effect; open/close state for mega dropdown hover |
| `MobileNavDrawer` | YES | open/close drawer state; accordion expand state |
| `HeroHeadline` | YES | motion animation triggers on component mount (not scroll) — requires `useEffect` or `motion` on mount |
| `AnimatedCounter` | YES | `useMotionValue`, `useSpring`, `useTransform`, `useInView` from motion/react |
| `ServicesTabSection` | YES | `useState` for active tab; `AnimatePresence` from motion/react |
| `AnimatedRingStat` | YES | SVG path animation + counter; `useInView`, `useMotionValue` |
| `FeatureCheckList` | YES | `useInView` for staggered reveal trigger |
| `MiniMetricsCard` | YES | `useInView` for bar chart grow trigger |
| `AnimatedBarChart` | YES | spring animation requires motion/react client hook |
| `CaseStudyPreviewCard` | YES | `useMotionValue` for metric counter |
| **Everything else** | NO | Server Components — no state, no browser APIs |

**Specifically NOT `"use client"`:**
- `HeroSection` (shell) — contains `HeroHeadline` as a child but is itself a Server Component
- `StatsBarSection` (shell) — contains `AnimatedCounter` children

## API Integration Contract

All data fetches in Phase 3 are build-time (SSG) or ISR-tagged. No `useEffect` data fetching.

| Component | Endpoint | Cache Tag | Fallback |
|-----------|----------|-----------|---------|
| `SiteNav` | `GET /api/nav-items` | `nav-items` | Empty arrays for non-Service panels |
| `SiteFooter` (links) | `GET /api/footer-links` | `footer-links` | Hardcoded column links |
| `SiteFooter` (settings) | `GET /api/settings` | `settings` | Hardcoded company info from CONTEXT.md |
| `StatsBarSection` | `GET /api/settings` | `settings` | Hardcoded seed values (150+, 50+, 6+, 98%) |

**Fetch helper to add to `src/lib/api.ts`:**
```ts
export async function fetchNavItems(): Promise<NavItem[]> {
  return fetchFromApi<NavItem[]>('/api/nav-items', {
    next: { tags: ['nav-items'] }
  }).catch(() => [])
}

export async function fetchFooterLinks(): Promise<FooterLink[]> {
  return fetchFromApi<FooterLink[]>('/api/footer-links', {
    next: { tags: ['footer-links'] }
  }).catch(() => [])
}

export async function fetchSettings(): Promise<Settings> {
  return fetchFromApi<Settings>('/api/settings', {
    next: { tags: ['settings'] }
  }).catch(() => SETTINGS_FALLBACK)
}
```

All helpers use `.catch(() => fallback)` — never throws to the component. Typed fallbacks prevent runtime errors.
