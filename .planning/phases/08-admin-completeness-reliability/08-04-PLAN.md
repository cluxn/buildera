# Plan 08-04: Partner Badges, Technologies & Popup Manager

## Objective

Create the PartnerBadge and Technology models + migrations + Filament resources (COMP-01). Add a Popup Manager page in Filament that lets the client configure exit-intent and idle popup settings (ADM-10 — the popup content is already stored in the Settings table via `popup_*` keys; the Popup Manager is a dedicated admin page for editing those settings in a UX-friendly way). Seed 6 sample partner badges (SEED-05).

## must_haves

```yaml
truths:
  - partner_badges table has: id, name, logo (nullable string for path), link (nullable), category, display_order, is_visible, timestamps
  - technologies table has: id, name, logo (nullable string for path), category (enum: Frontend/Backend/Cloud/CRM/E-Commerce), display_order, timestamps
  - PartnerBadgeResource is in "Website" navigation group with icon heroicon-o-shield-check
  - TechnologyResource is in "Website" navigation group with icon heroicon-o-cpu-chip
  - Popup Manager is a Filament Page (not a Resource) in the "Website" navigation group that reads/writes popup_* settings keys via Setting::set()
  - 6 partner badges seeded: Salesforce Partner, Google Cloud, Microsoft Solution Partner, AWS Partner, Shopify Partner, Meta Business Partner — each with category='platform', display_order 1-6, is_visible=true, logo=null (client uploads real logos)
  - PartnerBadge seeder is registered in DatabaseSeeder

goal_backward:
  - "Partner badge logos display on homepage, manageable from admin" (Success Criterion 5)
  - "Popup Manager lets admin configure exit-intent/idle popup content without code" (ADM-10)
```

```
class PopupManagerPage extends Page
{
    protected static string $view = 'filament.pages.popup-manager';
    protected static ?string $navigationGroup = 'Website';
    protected static ?string $navigationLabel = 'Popup Manager';
    protected static ?string $navigationIcon = 'heroicon-o-bell-alert';
    protected static ?int $navigationSort = 7;

    public ?array $data = [];

    public function mount(): void
    {
        $this->form->fill([
            'popup_exit_enabled'   => Setting::get('popup_exit_enabled', false),
            'popup_exit_headline'  => Setting::get('popup_exit_headline', ''),
            'popup_exit_subtext'   => Setting::get('popup_exit_subtext', ''),
            'popup_exit_cta'       => Setting::get('popup_exit_cta', ''),
            'popup_idle_enabled'   => Setting::get('popup_idle_enabled', false),
            'popup_idle_headline'  => Setting::get('popup_idle_headline', ''),
            'popup_idle_subtext'   => Setting::get('popup_idle_subtext', ''),
        ]);
    }

    public function form(Schema $schema): Schema
    {
        return $schema->statePath('data')->components([
            Section::make('Exit-Intent Popup')->schema([
                Toggle::make('popup_exit_enabled')->label('Enabled'),
                TextInput::make('popup_exit_headline')->label('Headline'),
                Textarea::make('popup_exit_subtext')->label('Body text')->rows(2),
                TextInput::make('popup_exit_cta')->label('CTA button text'),
            ]),
            Section::make('Idle Popup')->schema([
                Toggle::make('popup_idle_enabled')->label('Enabled'),
                TextInput::make('popup_idle_headline')->label('Headline'),
                Textarea::make('popup_idle_subtext')->label('Body text')->rows(2),
            ]),
        ]);
    }

    public function save(): void
    {
        $state = $this->form->getState();
        foreach ($state as $key => $value) {
            Setting::set($key, $value);
        }
        Notification::make()->title('Popup settings saved')->success()->send();
    }
}
```

Create the Blade view at `resources/views/filament/pages/popup-manager.blade.php`:

<x-filament-panels::page>
    <form wire:submit="save">
        {{ $this->form }}
        <x-filament::button type="submit" class="mt-4">Save Settings</x-filament::button>
    </form>
    <x-filament-actions::modals />
</x-filament-panels::page>
```

<?php

namespace Database\Seeders;

use App\Models\PartnerBadge;
use Illuminate\Database\Seeder;

class PartnerBadgeSeeder extends Seeder
{
    public function run(): void
    {
        $badges = [
            ['name' => 'Salesforce Partner', 'category' => 'platform', 'link' => 'https://www.salesforce.com', 'display_order' => 1],
            ['name' => 'Google Cloud Partner', 'category' => 'platform', 'link' => 'https://cloud.google.com', 'display_order' => 2],
            ['name' => 'Microsoft Solution Partner', 'category' => 'certification', 'link' => 'https://microsoft.com', 'display_order' => 3],
            ['name' => 'AWS Partner', 'category' => 'platform', 'link' => 'https://aws.amazon.com', 'display_order' => 4],
            ['name' => 'Shopify Partner', 'category' => 'marketplace', 'link' => 'https://www.shopify.com', 'display_order' => 5],
            ['name' => 'Meta Business Partner', 'category' => 'platform', 'link' => 'https://business.facebook.com', 'display_order' => 6],
        ];

        foreach ($badges as $badge) {
            PartnerBadge::firstOrCreate(
                ['name' => $badge['name']],
                array_merge($badge, ['is_visible' => true, 'logo' => null])
            );
        }
    }
}
```

Register in `DatabaseSeeder.php` — add `$this->call(PartnerBadgeSeeder::class)` to the `run()` method.

Run the seeder to populate the database: `php artisan db:seed --class=PartnerBadgeSeeder`
