# Plan 08-01: Unified Lead Inbox & Newsletter CSV Export

## Objective

Enhance the existing LeadResource with all filter options (date range, service interest, source_form with all values), a visible duplicate badge, proper status badge colors, and a notes field. Add CSV bulk export to NewsletterSubscriberResource so the client can download the full subscriber list.

## must_haves

```yaml
truths:
  - LeadResource table shows is_duplicate as a red badge when true, not a boring boolean icon
  - LeadResource filters include: source_form (SelectFilter), status (SelectFilter), is_duplicate (TernaryFilter), and a DateRange filter on submitted_at
  - LeadResource edit form has status (Select with new/contacted/qualified/lost) and notes (Textarea) as editable fields; all other fields disabled
  - NewsletterSubscriberResource has a bulk action "Export CSV" that downloads email,name,status,subscribed_at as a CSV
  - Both resources remain in the "Leads & CRM" navigation group
  - No new migrations required — existing leads and newsletter_subscribers tables have all needed columns

goal_backward:
  - "All leads from all form sources appear in unified inbox with correct source_form tag" (Success Criterion 1)
  - "Newsletter subscribers bulk-exportable as CSV" (ADM-03)
```

<action>
In NewsletterSubscriberResource.php, add a `->bulkActions([])` block to the table() method with one action:

```php
use Filament\Tables\Actions\BulkAction;
use Illuminate\Database\Eloquent\Collection;

BulkAction::make('export_csv')
    ->label('Export CSV')
    ->icon('heroicon-o-arrow-down-tray')
    ->action(function (Collection $records) {
        $csv = "email,name,status,subscribed_at\n";
        foreach ($records as $record) {
            $csv .= implode(',', [
                '"' . str_replace('"', '""', $record->email) . '"',
                '"' . str_replace('"', '""', ($record->name ?? '')) . '"',
                '"' . $record->status . '"',
                '"' . ($record->subscribed_at ?? '') . '"',
            ]) . "\n";
        }
        return response()->streamDownload(
            fn () => print($csv),
            'subscribers-' . now()->format('Y-m-d') . '.csv',
            ['Content-Type' => 'text/csv']
        );
    })
    ->requiresConfirmation()
    ->deselectRecordsAfterCompletion()
```

Add the necessary `use` statements at top of file. The `response()->streamDownload()` pattern works in Filament v5 bulk actions.
</action>
