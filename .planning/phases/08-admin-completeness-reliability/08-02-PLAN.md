# Plan 08-02: Nav & Footer Filament Editors

## Objective

Create NavItemResource and FooterLinkResource Filament resources so the client can manage navigation items and footer links from the admin panel — add, edit, reorder by display_order, toggle is_visible — without touching code. The NavItem and FooterLink models, migrations, and API controllers already exist; this plan builds only the Filament admin UI.

## must_haves

```yaml
truths:
  - NavItemResource is in the "Website" navigation group, icon heroicon-o-bars-3
  - NavItemResource form has: label (TextInput required), url (TextInput required), parent_group (Select with options: Services/Solutions/Work/Resources/standalone), display_order (TextInput numeric), is_visible (Toggle), opens_new_tab (Toggle)
  - NavItemResource table shows label, url, parent_group (badge), display_order, is_visible (icon) sorted by display_order asc
  - FooterLinkResource is in the "Website" navigation group, icon heroicon-o-link
  - FooterLinkResource form has: label (TextInput required), url (TextInput required), column_group (Select with options: Services/Solutions/Company/Resources), display_order (TextInput numeric), is_visible (Toggle)
  - FooterLinkResource table shows label, url, column_group (badge), display_order, is_visible (icon) sorted by display_order asc
  - Both resources support full CRUD (list, create, edit, delete)
  - No new migrations required — nav_items and footer_links tables already exist

goal_backward:
  - "Removing a nav item in Filament removes it from site nav without code deploy" (Success Criterion 2)
  - Admin can add/remove/reorder any nav link or dropdown group without code deploy (NAV-05)
```

<action>
Create `buildera-backend/app/Filament/Resources/NavItemResource.php` with:

```
namespace: App\Filament\Resources
model: App\Models\NavItem
navigationGroup: 'Website'
navigationLabel: 'Navigation Items'
navigationIcon: 'heroicon-o-bars-3'
navigationSort: 1

form() — Schema::make()->components([
    TextInput::make('label')->required()->maxLength(255),
    TextInput::make('url')->required()->maxLength(500)->placeholder('/services or https://...'),
    Select::make('parent_group')->options([
        'Services' => 'Services', 'Solutions' => 'Solutions',
        'Work' => 'Work', 'Resources' => 'Resources', 'standalone' => 'Standalone (no group)',
    ])->nullable(),
    TextInput::make('display_order')->numeric()->default(0),
    Toggle::make('is_visible')->default(true),
    Toggle::make('opens_new_tab')->default(false),
])

table() — Table::make()->columns([
    TextColumn::make('label')->searchable()->sortable(),
    TextColumn::make('url')->limit(40)->searchable(),
    TextColumn::make('parent_group')->badge()->sortable(),
    TextColumn::make('display_order')->sortable(),
    IconColumn::make('is_visible')->boolean(),
])->defaultSort('display_order')->actions([EditAction, DeleteAction])->bulkActions([DeleteBulkAction])

getPages():
    'index' => ListNavItems::route('/')
    'create' => CreateNavItem::route('/create')
    'edit' => EditNavItem::route('/{record}/edit')
```

Create the 3 page classes in `NavItemResource/Pages/`:
- `ListNavItems` extends `ListRecords`
- `CreateNavItem` extends `CreateRecord`
- `EditNavItem` extends `EditRecord`

Use `Filament\Schemas\Schema` (v5 API). Import all components from `Filament\Forms\Components\*` and `Filament\Tables\*`.
</action>

<acceptance_criteria>
- `php artisan about` runs with no errors
- NavItemResource appears in Filament admin under "Website" navigation group as "Navigation Items"
- Clicking "New navigation item" opens a form with all 6 fields
- Creating a nav item with label="Our Work", url="/work", parent_group="Work", is_visible=true saves to nav_items table
- The nav items list table shows the new item, sorted by display_order
- Editing and deleting nav items work
</acceptance_criteria>
</task>
