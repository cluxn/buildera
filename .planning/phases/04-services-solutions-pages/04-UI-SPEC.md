## Spacing Scale

Inherits the Phase 3 spacing scale exactly. No new tokens.

| Token | px Value | Tailwind Class | Usage |
|-------|----------|----------------|-------|
| xs | 4px | `gap-1`, `p-1` | Icon internal gaps, badge padding |
| sm | 8px | `gap-2`, `p-2` | Compact inline padding, icon-to-text gaps |
| md | 16px | `gap-4`, `p-4` | Default card padding, form field gaps |
| lg | 24px | `gap-6`, `p-6` | Card padding (desktop), nav item spacing |
| xl | 32px | `gap-8`, `p-8` | Section internal padding (mobile) |
| 2xl | 48px | `gap-12`, `py-12` | Section padding (desktop top/bottom) |
| 3xl | 64px | `gap-16`, `py-16` | Major section breaks (hero, CTA) |

**Phase 4 spacing exceptions:**

- Service hero section: `py-20 md:py-24` — matches homepage hero breathing room; must not feel cramped on large screens.
- Solution hero section: `py-16 md:py-20` — slightly less tall than service hero (no ring stat occupying vertical space).
- Process section (SequentialSteps): `gap-8` between steps vertically; connector line `h-8` between step circles.
- TechShowcase categories: `space-y-6` between category groups; `gap-3` between individual tech badges within a group.
- Service/solution page section horizontal padding: `px-4` (mobile) → `px-6` (sm) → `container mx-auto px-8` (lg+). Max content width: `max-w-7xl`. Same as Phase 3.
- Breadcrumb strip: `py-3 px-4` — compact; sits between nav and hero.

## Animation Contract

### Phase 4 New Animations (extending Phase 3 contract)

All Phase 3 animation contracts carry forward as-is. Phase 4 adds four new contracts:

---

#### Animation 1: AnimatedRingStat (Service Hero, `"use client"`)

**Trigger:** `whileInView` + `viewport={{ once: true }}` — service hero is above fold on some viewport sizes so use `useInView` with ref, not just `whileInView`.

**SVG parameters (locked — match Phase 3 Why Buildera Column 1 spec):**
```
r=52, cx=60, cy=60, stroke-width=8
circumference ≈ 326.7
```

```tsx
// Background track ring
<circle cx="60" cy="60" r="52" fill="none"
  stroke="hsl(217 91% 60% / 20%)" strokeWidth="8" />

// Animated arc
<motion.circle
  cx="60" cy="60" r="52"
  fill="none"
  stroke="var(--brand-primary)"
  strokeWidth="8"
  strokeLinecap="round"
  strokeDasharray="326.7"
  initial={{ pathLength: 0, opacity: 0 }}
  animate={isInView ? { pathLength: 1, opacity: 1 } : {}}
  transition={{ duration: 1.5, ease: 'easeInOut', delay: 0.3 }}
  style={{ rotate: -90, transformOrigin: 'center' }}
/>
```

Counter text inside ring: `foreignObject x="20" y="35" width="80" height="50"`.
Counter value: same `useMotionValue + useSpring + useTransform` pattern as Phase 3 `AnimatedCounter`.
Spring: `{ stiffness: 60, damping: 20 }` — same spring as stats bar.

Size: `size={140}` on desktop; `size={100}` on mobile — responsive via `w-[100px] md:w-[140px]` on wrapper.

**Reduced motion:** Gate `isInView` animation behind `useReducedMotion()` — if `prefersReducedMotion`, show static value + full ring immediately.

---

#### Animation 2: StaggeredRevealGrid (Outcome Cards, `"use client"`)

Used for: Service Outcome Cards (80ms stagger), Solution Feature Cards (80ms stagger), sub-service cards on category pages (80ms stagger).

```tsx
// Per card:
initial={{ opacity: 0, y: 24 }}
whileInView={{ opacity: 1, y: 0 }}
viewport={{ once: true, margin: '-100px' }}
transition={{
  delay: index * 0.08,  // 80ms per card
  duration: 0.5,
  ease: 'easeOut',
}}
```

Hover on each card: `whileHover={{ y: -4 }}` + CSS `box-shadow: 0 0 0 2px var(--brand-primary)` via class. Duration 0.2s.

**Reduced motion:** If `prefersReducedMotion`, render children without animation wrapper (static render).

---

#### Animation 3: SequentialSteps (Process Section, `"use client"`)

**Narrative:** Each step unlocks the next — connector line draws as next step reveals.

```tsx
// Each step card:
initial={{ opacity: 0, x: -16 }}
animate={isInView ? { opacity: 1, x: 0 } : {}}
transition={{ delay: stepIndex * 0.12, duration: 0.4, ease: 'easeOut' }}
// 120ms per step

// Connector line between steps (not after last):
initial={{ scaleY: 0, originY: 0 }}  // vertical on mobile
animate={isInView ? { scaleY: 1 } : {}}
transition={{ delay: stepIndex * 0.12 + 0.4, duration: 0.3, ease: 'easeOut' }}
// Line appears 400ms after its preceding step — after step text is readable
```

Desktop connector: horizontal line using `scaleX: 0 → 1` instead of `scaleY`.

**Single trigger:** entire `SequentialSteps` component uses one `useInView(ref, { once: true, margin: '-80px' })` — all steps triggered from a single ref on the parent div. Steps cascade via delay, not independent observers.

---

#### Animation 4: TechStaggerGrid (TechShowcase, `"use client"`)

Category header label: base reveal (`opacity 0 → 1, y: 12 → 0`), `delay: catIndex * 0.1`.

Per logo/badge within category: `scale: 0.9 → 1, opacity: 0 → 1`, `delay: categoryStartDelay + logoIndex * 0.06`. **60ms per logo.**

```tsx
initial={{ opacity: 0, scale: 0.9 }}
whileInView={{ opacity: 1, scale: 1 }}
viewport={{ once: true }}
transition={{
  delay: categoryStartDelay + i * 0.06,
  duration: 0.3,
  ease: 'easeOut',
}}
```

`categoryStartDelay` for category N: `N * 0.1 + 0.1` — so category 0 starts at 0.1s, category 1 at 0.2s, etc.

## `"use client"` Boundary Map

**Rule from CLAUDE.md:** `"use client"` ONLY on components requiring browser APIs or React state/effects.

| Component | Needs `"use client"` | Reason |
|-----------|---------------------|--------|
| `AnimatedRingStat` | YES | `useInView`, `useMotionValue`, `useSpring`, `useTransform` |
| `StaggeredRevealGrid` | YES | `useReducedMotion`, `whileInView` on children array |
| `SequentialSteps` | YES | `useInView` on parent ref for shared trigger; connector animation |
| `TechStaggerGrid` | YES | `whileInView` on individual logo badges |
| **Everything else** | NO | Server Components — no state, no browser APIs |

**Specifically NOT `"use client"` in Phase 4:**
- `ServiceDetailLayout` — shell only; animated children are leaf nodes
- `SolutionDetailLayout` — same
- All section components (`ServiceHero`, `ServiceProcess`, etc.) — they import animated leaf nodes but are themselves Server Components
- `TechShowcase` — Server Component shell; `TechStaggerGrid` is the `"use client"` child
- `Breadcrumb` — pure JSX, no state
- `VideoEmbed` — conditional iframe; no browser state
- `ServiceFaq` — uses shadcn `Accordion` which is itself a `"use client"` component (its own boundary); `ServiceFaq` does not need `"use client"` at the wrapper level

**Note on shadcn `Accordion`:** shadcn accordion components include their own `"use client"` boundary. `ServiceFaq.tsx` can remain a Server Component and import the shadcn `Accordion` — the boundary is contained inside the shadcn component file.
