<task id="09-05-T3" type="checkpoint:human-verify" gate="blocking">
<title>Cross-browser + cross-device verification</title>
<what-built>
All Phase 9 performance and accessibility fixes are in place and Lighthouse scores are recorded in PHASE-09-QA.md. The remaining verification (rendering correctness across browsers and devices, animation smoothness, Safari) requires human/manual testing because Safari is not available natively on Windows and visual layout correctness is not fully automatable.
</what-built>
<how-to-verify>
1. With the production build running (http://localhost:3001), open the homepage, a service detail page, a blog post, /contact, and a 404 page (e.g. /nope) in: Chrome, Firefox, and Edge. Confirm no layout breaks and animations are smooth.
2. For Safari: use BrowserStack free tier (research notes ~100 min/month) — load the same pages; confirm rendering and that backdrop-filter glassmorphism and motion animations degrade gracefully.
3. Device widths via Chrome DevTools device toolbar (or real devices): iPhone SE (375px — smallest), iPad (768px), 1080p desktop (1920px). Confirm: nav/mobile drawer works, no horizontal scroll, touch targets are tappable, hero and grids reflow correctly.
4. Tab through the homepage and contact form with the keyboard — confirm the focus-visible brand ring (from 09-03) is clearly visible.
5. Fill in the Browser Matrix and Device Matrix in PHASE-09-QA.md with PASS or the issue found for each cell.
</how-to-verify>
<resume-signal>Type "approved" when all browser/device cells are PASS (or remaining issues are logged and accepted), or describe the issues to fix.</resume-signal>
