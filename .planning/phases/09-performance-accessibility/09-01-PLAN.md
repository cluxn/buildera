- buildera-frontend/src/components/layout/MegaDropdown.tsx (product logo Image — confirm sizes; add matching pixel sizes if missing)
</read_first>
<action>
For each `<Image>` that lacks a `sizes` prop, add `sizes` equal to the exact rendered pixel width as a string: AuthorBio → `sizes="64px"`; TestimonialCard company logo → `sizes="40px"`; TestimonialsSection avatar (width 48) → `sizes="48px"` and avatar (width 36) → `sizes="36px"`; BlogDetailHero author avatar → `sizes="24px"` (only if absent); BlogPostCard author avatar → `sizes="20px"` (only if absent); MegaDropdown product logo → set to its rendered pixel width (only if absent). Leave responsive `fill` content images that already have a media-query `sizes` string untouched.
</action>

- `grep -n 'sizes="40px"' buildera-frontend/src/components/content/TestimonialCard.tsx` matches
- `grep -cE 'sizes="(48|36)px"' buildera-frontend/src/components/sections/TestimonialsSection.tsx` returns 2
